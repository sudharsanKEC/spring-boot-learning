What is an annotation in java?
	Annotation = metadata (information) added to code
		It does not change logic directly
		It gives extra information to:
			Compiler
			JVM
			Tools / Frameworks
	Annotations are a form of metadata in Java used to provide additional information about classes, methods, or variables, which can be processed at compile-time or runtime.

What problem it solves and what existed before it?
	Before annotations (Old Spring style)
	Everything was configured in XML files:
		<bean id="userService" class="com.example.UserService">
		    <property name="repo" ref="userRepository"/>
		</bean>
	Problems:
		Too much boilerplate
		Hard to read
		Hard to maintain
		Separate config file (not near code)
		Easy to make mistakes
	
	With annotations
	Same thing becomes:
		@Service
		public class UserService {
		    @Autowired
		    private UserRepository repo;
		}
	Benefits:
		Less code
		Easy to understand
		Configuration is close to the code
		Faster development

What annotations really do in Spring
	In Spring:
		Annotations tell the Spring container how to behave
	Spring reads them using:
		Reflection
		Classpath scanning

Example (Real Understanding):
	@Service
	class UserService {
	}
What happens?
	Spring scans this class
	Sees @Service
	Creates an object (bean)
	Stores it in ApplicationContext

How Spring uses annotations internally
Flow:
	Spring starts
	Scans packages
	Finds annotations (@Component, etc.)
	Uses reflection
	Creates beans
	Manages them

Important Spring Annotations (Grouped)
	Bean Creation
		@Component → generic bean
		@Service → business logic
		@Repository → database layer
		@Controller / @RestController → web layer
	All of these = “Create this as a bean”

	Dependency Injection
		@Autowired → inject dependency
		@Qualifier → choose specific bean
	Configuration
		@Configuration → config class
		@Bean → manually define bean
	Web / API
		@RequestMapping
		@GetMapping, @PostMapping
	Used by DispatcherServlet to map requests

	Spring Boot Special
		@SpringBootApplication
		Combines:
			@Configuration
			@EnableAutoConfiguration
			@ComponentScan

Note: Annotations don't do anything on their own, they works only because spring reads them and spring decides what actions to take on the respective annotation.

Purpose of annotations:
	Reduce XML configuration
	Simplify development
	Enable declarative programming


When does Spring scan annotations?
	During application startup
	When this runs:
		SpringApplication.run(App.class, args);
	Spring does:
		Create ApplicationContext
		   ↓
		Scan packages (Component Scan)
		   ↓
		Find annotated classes
		   ↓
		Create beans
		   ↓
		Store in container

Which phase exactly?
	Inside ApplicationContext initialization phase
	Before your app starts accepting requests
	After dependencies are loaded
	During container setup
	So:
		Annotation scanning happens at startup, not runtime per request

How Spring scans annotations?
	Step-by-step:
		Starts from your main class:
			@SpringBootApplication
		This includes:
			@ComponentScan
			Spring scans:
				Current package
				Sub-packages
			For each class:
				Uses reflection(Reflection is a feature in Java that allows a program to: Inspect and manipulate classes, methods, fields, and annotations at runtime
						Provided by core Java (java.lang.reflect), used heavily by Spring Framework.)
			Checks for annotations like:
				@Component
				@Service
				etc.
			If found:
				Creates object (bean)
				Stores in ApplicationContext

The entire process of scanning the package and sub packages is called component scan.
The scanning is done primarily for processing annotations.
The scanning is done for every java files in com.example.spring_app and the scanning is done in its sub packages also.
But if a java class is outside of it then we want to explicitly made it scan by spring.

Difference between these annotations
Important truth:
	All of these are specialized versions of @Component
	@Component (Base)
		Generic bean
		Used for any class
	@Service
		Used for business logic layer
		Just semantic (no major functional difference)
	@Repository
		Used for DAO layer
		Adds:
			Exception translation (important)
	@Controller
		Used for web layer (MVC)
		Works with DispatcherServlet

Your Question (Very Important)
	If a class has these annotations, can we directly use it for DI?
	YES
	Example:
		@Service
		class UserService {}
		@RestController
		class UserController {
			@Autowired
			UserService service;
		}
	Why it works?
		Spring already created UserService bean
		It injects it automatically

What about @Bean?
	This is DIFFERENT from above annotations
	@Bean is used for manual bean creation
	
	@Configuration
	class AppConfig {
	    @Bean
	    public UserService userService() {
	        return new UserService();
	    }
	}
	When do we use @Bean?
	When:
		You can’t modify the class
		Third-party class
		Need custom configuration

| Annotation   | Usage               |
| ------------ | ------------------- |
| `@Component` | Automatic detection |
| `@Bean`      | Manual creation     |


@Component / @Service / @Repository / @Controller
   → Auto detected

@Bean
   → Manually defined
