What is Spring Security?
	Spring Security is a powerful security framework for Java applications, especially Spring Boot, that provides built-in tools to secure your application at multiple layers.
	It provides security features for:
		1) Authentication
			Verifies who the user is
			Login forms, HTTP Basic, OAuth2, JWT, etc.
		2) Authorization
			Verifies what the user can access
			Role-based access, method-level security (@PreAuthorize, @Secured)
		3) Password Security
			Hashing + salting (BCryptPasswordEncoder)
			Protects stored passwords
		4) Password Security
			Hashing + salting (BCryptPasswordEncoder)
			Protects stored passwords
		5) CSRF Protection
			Prevents cross-site request forgery attacks automatically
		6) Clickjacking Protection
			Adds X-Frame-Options headers automatically
		7) CORS Handling
			Lets you safely allow cross-origin requests for your frontend
		8) Other Protections
			Brute-force prevention (with lockouts or throttling)
			Security headers (X-Content-Type-Options, X-XSS-Protection)

How It Works in Spring Boot?
	Spring Security integrates seamlessly into Spring Boot apps.
	By default, it applies basic security:
		All endpoints secured
		Default login form
		CSRF protection enabled
		You can customize easily using WebSecurityConfigurerAdapter or SecurityFilterChain.
	Spring Security = built-in shield for your Spring Boot app against authentication, authorization, session, CSRF, clickjacking, password, and many web attacks
	
To add spring security to our spring boot app,
	i) Goto POM.xml, add this dependency,
			<dependency>
			    <groupId>org.springframework.boot</groupId>
			    <artifactId>spring-boot-starter-security</artifactId>
			</dependency>
		Just adding this is enough and everything will be auto configured by spring boot.
		It adds all the basic default security to all endpoints, after adding spring security, you cannot directly access the endpoints until we configure it manually.
		It shows a default login page.
		The default username for logging in is user, and the password will be generated by spring boot in the terminal console logs. You can use that to login into your application

Why this happens?
	Because of Auto Configuration
	Spring Boot detects:
		spring-boot-starter-security
	Then automatically:
		Creates SecurityFilterChain
		Secures all endpoints
		Enables login form
		Enables HTTP Basic Auth

Internal Auto Setup:
	Without writing any code, Spring creates:
		Default SecurityFilterChain
		Default UserDetailsService
		Default PasswordEncoder
	That’s why everything works instantly

Before Adding Spring Security:
	Our app is completely open
	What happens?
		Client Request → Tomcat → DispatcherServlet → Controller → Response
	No security layer ❌
		Everything is PUBLIC
		Example APIs:
			GET    /users
			POST   /users
			DELETE /users/1
	Anyone can call:
		From browser
		From Postman
		From anywhere

	Problems
		No authentication ❌
		No authorization ❌
		Anyone can:
			Delete data 
			Modify users
			Access sensitive info
Before Spring Security:
| Feature        | Status    |
| -------------- | --------- |
| Authentication |   None    |
| Authorization  |   None    |
| API access     |    Public |
| Protection     |   No      |


AFTER Adding Spring Security
	Now your app becomes protected by default
	What changes in flow?
		Client Request
		   ↓
		Tomcat
		   ↓	
		Spring Security Filters
		   ↓
		DispatcherServlet
		   ↓
		Controller

Everything is LOCKED
	Same APIs:
		GET    /users
		POST   /users
		DELETE /users/1
	Now:
		Cannot access directly
		Must authenticate first

	What is required now?
		1. Authentication
			Username + Password
			Basic Auth / Login form
		2. Authorization (later when configured)
			Roles like ADMIN / USER

	What you’ll see
	In Postman:
		GET /users
		Response:
			401 Unauthorized
		In Browser:	
			Redirected to:
				Default login page
		Default Credentials
			Username: user
			Password: (from console log)

	Why Everything is Locked?
		Spring follows:
			“Secure everything by default”

| Feature        | Before Security | After Security  |
| -------------- | --------------- | --------------- |
| API Access     | Public 🌍       | Restricted 🔐   |
| Authentication | ❌               | ✔️ Required     |
| Authorization  | ❌               | ✔️ Possible     |
| Login System   | ❌               | ✔️ Auto         |
| Security Layer | ❌               | ✔️ Filter Chain |


Before:
	Office with no security
		Anyone can walk in
After:
	Office with security guard 
		Must show ID to enter

Note: By default, spring security add protection to all endpoints.
